# SNI Proxy Deploy

Автоматизированный комплекс для развертывания SNI-прокси сервера и обхода блокировок для ИИ-сервисов (и не только). 
Комплекс объединяет списки доменов из двух источников (`dns.malw.link` и `GeoHideDNS`) и автоматически перенаправляет неизвестный трафик в fallback-порт (для совместной работы с X-ray/VLESS).

## Что внутри:
* **SNI-прокси (Nginx)**: Работает на `443` порту. Если запрос летит к разрешенному ИИ-сайту, Nginx прозрачно проксирует его к оригиналу, обходя РКН.
* **X-ray Fallback**: Если запрос летит к неизвестному сайту (в том числе ваш замаскированный VPN-трафик), Nginx перекидывает его на `127.0.0.1:8443` к вашему X-ray.
* **GitHub Actions**: Ежедневно в 03:00 парсит и объединяет списки из двух источников, формируя `lists/domains.txt` для Nginx и `mega_hosts_template.txt` для ваших роутеров.
* **Авто-апдейт**: VPS и роутеры ежедневно по крону подтягивают свежие списки из этого репозитория. Никакой ручной работы!

## 🚀 Инструкция: Как развернуть на новом сервере (VPS)

1. Зайдите на новый VPS-сервер:
   ```bash
   ssh root@ВАШ_IP
   ```
2. Склонируйте этот репозиторий:
   ```bash
   git clone https://github.com/uraz0v/sni-proxy.git /opt/sni-proxy
   ```
3. Запустите скрипт установки:
   ```bash
   cd /opt/sni-proxy
   chmod +x deploy.sh
   ./deploy.sh
   ```
> Скрипт сам установит нужный часовой пояс (Екатеринбург), скачает Docker, загрузит актуальные домены, поднимет прокси и настроит ежедневное автообновление в `cron`.
> **Важно**: Ваш X-ray (или другой VPN) должен быть настроен на прослушивание порта `8443`, чтобы ловить весь остальной трафик.

## 🌐 Инструкция: Как настроить роутер OpenWrt

Чтобы ваши устройства дома прозрачно использовали этот прокси для ИИ-сервисов, необходимо обновить файл `hosts` на роутере.

Мы подготовили умный скрипт, который делает всё сам: он скачивает сгенерированный на GitHub шаблон, подставляет IP вашего VPS, делает бекап оригинального файла `hosts` (чтобы ничего не сломать) и добавляет себя в `cron` для ежедневного автообновления.

1. Зайдите на ваш роутер по SSH:
   ```bash
   ssh root@192.168.1.1
   ```
2. Скачайте и запустите скрипт автообновления (используем `curl` для обхода проблем со старыми версиями `wget`):
   ```bash
   curl -sL -o /etc/update_router_hosts.sh https://raw.githubusercontent.com/uraz0v/sni-proxy/main/update_router_hosts.sh && sh /etc/update_router_hosts.sh ВАШ_VPS_IP
   ```
   *(Например: `... && sh /etc/update_router_hosts.sh 1.2.3.4`)*

Всё! Скрипт проверит изменения, обновит `hosts`, перезапустит Dnsmasq и добавит задачу в `cron`, чтобы делать это каждую ночь в 05:00. Никаких конфликтов, и ваши кастомные записи (до комментария `# --- SNI PROXY AUTOGENERATED ---`) никогда не затрутся!

## 📁 Структура проекта (Что делает каждый файл)

* **`deploy.sh`** — скрипт первичной установки для вашего VPS. Настраивает часовой пояс, устанавливает Docker, запускает прокси и добавляет задание в cron.
* **`docker-compose.yml`** — конфигурация Docker для запуска Nginx-прокси. Пробрасывает 443 порт сервера и часовые пояса внутрь контейнера.
* **`nginx.conf`** — сердце проксирования. Принимает трафик, читает SNI домена. Если домен есть в `domains.txt`, перенаправляет прозрачно (как TCP stream). Если домен неизвестен — кидает трафик на `127.0.0.1:8443` к вашему X-ray.
* **`update_lists.sh`** — скрипт, который каждую ночь запускается на VPS (в cron), делает `git pull` и перезагружает Nginx, если в списках появились новые домены. Записывает красивые логи с датой в `/var/log/sni-proxy-update.log`.
* **`entrypoint.sh`** — внутренний стартовый скрипт Docker-контейнера. Он преобразует простые списки доменов в формат Nginx (регулярные выражения) прямо перед запуском.
* **`.github/workflows/update.yml`** — конфигурация GitHub Actions. Автоматически запускает сборку списков каждое утро.
* **`generate.sh`** — парсер списков. Качает домены из исходников, склеивает их, генерирует `lists/domains.txt` и `mega_hosts_template.txt`. Имеет настройку `SERVICES_TO_PROXY="ALL"`, позволяющую выбрать конкретные сервисы.
* **`lists/domains.txt`** — единый сгенерированный список доменов для Nginx, по которому он понимает, кого пускать через прозрачный прокси.
* **`lists/available_services.txt`** — удобная шпаргалка, которая автоматически генерируется скриптом `generate.sh`. В ней перечислены все доступные блоки из GeoHide, чтобы вы могли легко выбрать, какие из них проксировать.
* **`mega_hosts_template.txt`** — сгенерированный шаблон для файла `hosts` на роутерах. Содержит тысячи доменов, где напротив каждого стоит `YOUR_VPS_IP`.
* **`update_router_hosts.sh`** — умный скрипт для роутера OpenWrt. Скачивает `mega_hosts_template.txt`, подставляет реальный IP вашего сервера, бережно делает резервную копию вашего локального `hosts` и прописывает себя в cron роутера.
